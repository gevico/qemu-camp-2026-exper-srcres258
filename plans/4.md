# Plan: 为 Xg233ai `expand` 指令实现 TCG 翻译

## 背景

`expand` 是 Xg233ai 自定义扩展中的一条指令（funct7 = `0110110` = 54），功能是将源数组中每个 8-bit 元素按 4-bit 拆分为两个 8-bit 元素。它是 `crush`（8-bit → 4-bit 压缩）的逆操作。

测试文件 `tests/gevico/tcg/riscv64/test-insn-expand.c` 已经存在，包含 `split_to_4bits`、`custom_expand`、`compare` 三个测试用例。只需在 TCG 前端补全指令实现即可通过测试。

## 实现方案

需要修改 4 个文件，每个文件加一行/一个函数，完全遵循 `crush` 指令的已有模式：

### Step 1: 在 `xg233ai.decode` 中添加 `expand` 指令的译码规则

**文件**: `target/riscv/xg233ai.decode`

在 `crush` 行之后添加：

```decode
expand   0110110  ..... ..... 110 ..... 1111011  @r
```

funct7 = `0110110`（十进制 54），与 `crush` 的 `0100110`（38）格式一致，仅 funct7 不同。

### Step 2: 在 `helper.h` 中声明 `expand` 的 helper 函数

**文件**: `target/riscv/helper.h`

在 `crush` 的 `DEF_HELPER_FLAGS_4` 行之后添加：

```c
DEF_HELPER_FLAGS_4(expand, TCG_CALL_NO_WG, void, env, tl, tl, tl)
```

参数签名为 `(env, rd_addr, rs1_addr, rs2_count)`，与 `crush` 完全一致。

### Step 3: 在 `op_helper.c` 中实现 `expand` 的 helper 函数

**文件**: `target/riscv/op_helper.c`

在 `HELPER(crush)` 函数之后添加 `HELPER(expand)` 实现：

```c
void HELPER(expand)(CPURISCVState *env, target_ulong rd,
                    target_ulong rs1, target_ulong rs2)
{
    int n = (int)rs2;

    for (int i = 0; i < n; i++) {
        uint8_t val = cpu_ldub_data(env, rs1 + i);
        cpu_stb_data(env, rd + 2 * i, val & 0x0F);
        cpu_stb_data(env, rd + 2 * i + 1, (val >> 4) & 0x0F);
    }
}
```

逻辑直接对应伪代码：对源数组每个字节，低 4 位写入 `dst[2*i]`，高 4 位写入 `dst[2*i+1]`。使用 `cpu_ldub_data` / `cpu_stb_data` 进行 8-bit 内存访问。

### Step 4: 在 `trans_xg233ai.c.inc` 中添加 `expand` 的翻译函数

**文件**: `target/riscv/insn_trans/trans_xg233ai.c.inc`

在 `trans_crush` 函数之后添加：

```c
static bool trans_expand(DisasContext *ctx, arg_r *a)
{
    if (!ctx->cfg_ptr->ext_xg233ai) {
        return false;
    }

    TCGv rd = get_gpr(ctx, a->rd, EXT_NONE);
    TCGv rs1 = get_gpr(ctx, a->rs1, EXT_NONE);
    TCGv rs2 = get_gpr(ctx, a->rs2, EXT_NONE);

    gen_helper_expand(tcg_env, rd, rs1, rs2);
    return true;
}
```

与 `trans_crush` 结构完全相同，仅调用不同的 helper。
