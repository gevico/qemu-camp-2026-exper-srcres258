# Plan: 实现 `crush` 指令的 TCG 翻译

## 目标

为 Xg233ai 自定义扩展中的 `crush` 指令实现 QEMU TCG 前端翻译，使测试 `test-insn-crush` 通过。

## 指令规格

- **编码**: `0100110 rs2 rs1 110 rd 1111011`（R-type，opcode=0x7b，funct3=6，funct7=38）
- **功能**: 将源数组每个 UINT8 元素的低 4 位两两打包为一个 UINT8 元素
- **操作数**: `rd`=目标地址, `rs1`=源地址, `rs2`=元素数量 N

## 现有代码分析

测试文件 `tests/gevico/tcg/riscv64/test-insn-crush.c` 已存在，包含完整测试逻辑。当前缺少 TCG 前端实现，指令会触发非法指令异常。

已有的类似指令 `dma` 和 `sort` 提供了清晰的实现模式，涉及 3 个文件的修改。

## 实现步骤

### 步骤 1: 在 `target/riscv/xg233ai.decode` 中添加指令编码

在文件末尾（`sort` 行之后）添加：

```
crush    0100110  ..... ..... 110 ..... 1111011  @r
```

`0100110` 是指令编码中的 funct7 高 7 位部分，与 `dma`（`0000110`）和 `sort`（`0010110`）处于同一 opcode 空间。decodetree 工具会据此生成 `decode_crush` 函数。

### 步骤 2: 在 `target/riscv/helper.h` 中声明 helper 函数

在 `/* Xg233ai helpers */` 区域（第 1290 行 `DEF_HELPER_FLAGS_4(sort, ...)` 之后）添加：

```c
DEF_HELPER_FLAGS_4(crush, TCG_CALL_NO_WG, void, env, tl, tl, tl)
```

参数依次为：`env`（CPU 环境）、`rd`（目标地址）、`rs1`（源地址）、`rs2`（元素数量）。

### 步骤 3: 在 `target/riscv/insn_trans/trans_xg233ai.c.inc` 中添加 TCG 翻译器

在文件末尾（`trans_sort` 之后）添加：

```c
static bool trans_crush(DisasContext *ctx, arg_r *a)
{
    if (!ctx->cfg_ptr->ext_xg233ai) {
        return false;
    }

    TCGv rd = get_gpr(ctx, a->rd, EXT_NONE);
    TCGv rs1 = get_gpr(ctx, a->rs1, EXT_NONE);
    TCGv rs2 = get_gpr(ctx, a->rs2, EXT_NONE);

    gen_helper_crush(tcg_env, rd, rs1, rs2);
    return true;
}
```

### 步骤 4: 在 `target/riscv/op_helper.c` 中实现 helper 函数

在 `HELPER(sort)` 实现之后（第 812 行之后）添加：

```c
void HELPER(crush)(CPURISCVState *env, target_ulong rd,
                   target_ulong rs1, target_ulong rs2)
{
    int n = (int)rs2;

    for (int i = 0; i < n / 2; i++) {
        uint8_t lo = cpu_ldub_data(env, rs1 + 2 * i) & 0x0F;
        uint8_t hi = cpu_ldub_data(env, rs1 + 2 * i + 1) & 0x0F;
        cpu_stb_data(env, rd + i, lo | (hi << 4));
    }
    if (n & 1) {
        cpu_stb_data(env, rd + n / 2, cpu_ldub_data(env, rs1 + n - 1) & 0x0F);
    }
}
```

使用 `cpu_ldub_data` / `cpu_stb_data` 访问 guest 内存中的 UINT8 元素，与 `dma` 和 `sort` 中使用 `cpu_ldl_data` / `cpu_stl_data` 的模式一致。
