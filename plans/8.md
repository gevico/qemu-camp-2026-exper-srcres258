# 实现计划：Xg233ai `vmax` 指令翻译

## 概述

`vmax` 是 Xg233ai 自定义扩展中 10 条指令之一，当前已实现 7 条（dma, sort, crush, expand, vdot, vrelu, vscale），`vmax` 属于未实现的 3 条之一。

`vmax` 与 `vdot` 同属"归约操作"类别——从内存数组中计算一个标量结果并写回 `gpr[rd]`。因此实现模式与 `vdot` 完全一致（返回值为 `tl`，使用 `dest_gpr` + `gen_set_gpr`）。

测试文件 `tests/gevico/tcg/riscv64/test-insn-vmax.c` 已存在，无需修改。只需在 QEMU TCG 前端补全指令的译码、翻译和 helper 实现即可。

## 实现步骤

### 步骤 1：在 `xg233ai.decode` 添加译码条目

在文件末尾（第 19 行 `vscale` 之后）追加：

```
vmax     1110110  ..... ..... 110 ..... 1111011  @r
```

- `funct7 = 1110110`（十进制 118），这是 `vmax` 的唯一标识
- 其余字段（opcode `1111011`、funct3 `110`、寄存器位）与同扩展其他指令相同
- decodetree 工具会在构建时重新生成 `decode-xg233ai.c.inc`

### 步骤 2：在 `helper.h` 添加 helper 声明

在 Xg233ai helpers 区块（第 1295 行 `vscale` 之后）追加：

```c
DEF_HELPER_FLAGS_3(vmax, TCG_CALL_NO_WG, tl, env, tl, tl)
```

参数说明：
- 3 个参数（不含 env）：`rs1`（数组地址），`rs2`（元素数量）
- 返回类型 `tl`（target_ulong），因为结果要写回 `gpr[rd]`
- 与 `vdot` 使用相同的 `DEF_HELPER_FLAGS_3` 模式

### 步骤 3：在 `trans_xg233ai.c.inc` 添加翻译函数

在文件末尾（第 98 行 `trans_vscale` 之后）追加：

```c
static bool trans_vmax(DisasContext *ctx, arg_r *a)
{
    if (!ctx->cfg_ptr->ext_xg233ai) {
        return false;
    }

    TCGv rs1 = get_gpr(ctx, a->rs1, EXT_NONE);
    TCGv rs2 = get_gpr(ctx, a->rs2, EXT_NONE);
    TCGv rd = dest_gpr(ctx, a->rd);

    gen_helper_vmax(rd, tcg_env, rs1, rs2);
    gen_set_gpr(ctx, a->rd, rd);
    return true;
}
```

关键点：
- `rs1` = 源数组起始地址，`rs2` = 元素数量 N，通过 `get_gpr` 读取
- `rd` 通过 `dest_gpr` 分配目标 TCGv（处理 x0 和截断）
- `gen_helper_vmax` 的第一个参数是返回值（rd），然后是 `tcg_env`，最后是两个输入
- `gen_set_gpr` 将结果写回寄存器文件

### 步骤 4：在 `op_helper.c` 添加 helper 实现

在文件末尾（第 875 行 `HELPER(vscale)` 之后）追加：

```c
target_ulong HELPER(vmax)(CPURISCVState *env,
                          target_ulong rs1, target_ulong rs2)
{
    int n = (int)rs2;
    int32_t max = (int32_t)cpu_ldl_data(env, rs1);

    for (int i = 1; i < n; i++) {
        int32_t val = (int32_t)cpu_ldl_data(env, rs1 + i * 4);
        if (val > max) {
            max = val;
        }
    }

    return (target_ulong)(int32_t)max;
}
```

逻辑说明：
1. 从 `rs1` 地址读取第一个 INT32 元素作为初始最大值
2. 遍历数组剩余元素（`cpu_ldl_data` 以小端读取 32 位），更新最大值
3. 返回时通过 `(int32_t)` 再转 `(target_ulong)` 完成 32 位到 64 位的符号扩展
