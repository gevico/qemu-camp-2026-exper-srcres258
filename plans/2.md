# Plan: 为 Xg233ai `sort` 指令实现 TCG 指令翻译

## 目标

在 QEMU TCG 前端为 Xg233ai 自定义扩展中的 `sort` 指令实现完整的指令翻译流程，使其通过 `test-insn-sort` 测试。

## 指令规格

- **助记符**: `sort`
- **功能**: 对内存中的 INT32 数组执行升序冒泡排序（部分排序），结果原地写回
- **编码**: R-type, opcode=`1111011`(0x7B), funct3=`110`, funct7=`0010110`(22)
- **操作数映射**（来自测试文件 `custom_sort` 的内联汇编）:
  - `rd` → 参与排序的元素数量 K
  - `rs1` → 数组在内存中的起始地址
  - `rs2` → 数组总元素数量 N（边界约束，排序算法仅使用 K）

## 现有参考实现

项目中已实现 `dma` 指令，可作为完整的参考模式。`dma` 的实现涉及以下 4 个文件：

1. `target/riscv/xg233ai.decode` — decodetree 译码定义
2. `target/riscv/insn_trans/trans_xg233ai.c.inc` — TCG 翻译函数
3. `target/riscv/helper.h` — Helper 函数声明
4. `target/riscv/op_helper.c` — Helper 函数实现

## 实现计划

### 步骤 1: 在 `xg233ai.decode` 中添加 sort 译码条目

**文件**: `target/riscv/xg233ai.decode`

在现有的 `dma` 条目之后，添加：

```
sort     0010110  ..... ..... 110 ..... 1111011  @r
```

原理：decodetree 根据 funct7 字段区分同一 opcode+funct3 空间内的不同指令。`0010110` 是 sort 的 funct7 编码（十进制 22）。`@r` 复用已定义的 R-type 格式模板。

### 步骤 2: 在 `trans_xg233ai.c.inc` 中添加 `trans_sort` 翻译函数

**文件**: `target/riscv/insn_trans/trans_xg233ai.c.inc`

在 `trans_dma` 之后添加 `trans_sort`，遵循完全相同的模式：

```c
static bool trans_sort(DisasContext *ctx, arg_r *a)
{
    if (!ctx->cfg_ptr->ext_xg233ai) {
        return false;
    }

    TCGv addr = get_gpr(ctx, a->rs1, EXT_NONE);
    TCGv total = get_gpr(ctx, a->rs2, EXT_NONE);
    TCGv count = get_gpr(ctx, a->rd, EXT_NONE);

    gen_helper_sort(tcg_env, addr, total, count);
    return true;
}
```

要点：
- 扩展检查守卫 `ext_xg233ai`，未启用时返回 `false`（指令非法）
- 通过 `get_gpr()` 读取三个寄存器操作数，`EXT_NONE` 不做符号/零扩展
- 将三个参数以 `addr, total, count` 顺序传给 helper（与 helper 签名对齐）

### 步骤 3: 在 `helper.h` 中声明 sort helper

**文件**: `target/riscv/helper.h`（约 1289 行，`/* Xg233ai helpers */` 区域）

在 `DEF_HELPER_FLAGS_4(dma, ...)` 之后添加：

```c
DEF_HELPER_FLAGS_4(sort, TCG_CALL_NO_WG, void, env, tl, tl, tl)
```

- `TCG_CALL_NO_WG`: 标记不写全局 guest 状态（允许 TCG 优化）
- 签名: `void helper_sort(env, addr, total, count)`

### 步骤 4: 在 `op_helper.c` 中实现 sort helper

**文件**: `target/riscv/op_helper.c`（约 797 行之后，dma helper 之后）

```c
void HELPER(sort)(CPURISCVState *env, target_ulong base,
                  target_ulong total, target_ulong count)
{
    for (int i = 0; i < count - 1; i++) {
        for (int j = 0; j < count - i - 1; j++) {
            int32_t a = (int32_t)cpu_ldl_data(env, base + j * 4);
            int32_t b = (int32_t)cpu_ldl_data(env, base + (j + 1) * 4);
            if (a > b) {
                cpu_stl_data(env, base + j * 4, (uint32_t)b);
                cpu_stl_data(env, base + (j + 1) * 4, (uint32_t)a);
            }
        }
    }
}
```

要点：
- 使用 `cpu_ldl_data` / `cpu_stl_data` 访问 guest 内存（与 dma helper 一致），自动处理 TLB、MMU 等
- `cpu_ldl_data` 返回 `uint32_t`，需显式转为 `int32_t` 以实现有符号比较
- 仅排序前 K 个元素（`count` 个），其余元素保持不变
- `count` 为 0 或 1 时循环不执行，行为正确
- `total` 参数（`rs2`）仅在伪代码中作为边界约束，排序算法本身不使用
