# 计划：实现 Xg233ai `vadd` 指令的 TCG 翻译

## 概述

`vadd` 是 Xg233ai 扩展中的第 10 条指令（funct7=30），实现 INT32[16] 向量逐元素加法。目前 QEMU 中已实现其余 9 条指令，`vadd` 缺失。测试文件 `tests/gevico/tcg/riscv64/test-insn-vadd.c` 已经存在，只需在 QEMU 侧补齐翻译和执行逻辑。

## 指令规格

- **编码**：R-type，opcode=0x7b，funct3=6，funct7=30（二进制 `0011110`）
- **操作**：从 `gpr[rs1]` 和 `gpr[rs2]` 地址各读 16 个 INT32 元素，逐元素相加，结果写入 `gpr[rd]` 地址
- **特性**：溢出按补码回绕；支持原地操作（rd 与 rs1 或 rs2 相同）

## 需要修改的文件（共 4 个）

### 1. `target/riscv/xg233ai.decode`（译码表）

在 `gemm` 条目之后添加一行：

```
vadd     0011110  ..... ..... 110 ..... 1111011  @r
```

将 `vadd` 注册到 decodetree 译码器中，funct7 模式 `0011110` 对应十进制 30。

### 2. `target/riscv/helper.h`（helper 声明）

在 Xg233ai helpers 区块末尾（`DEF_HELPER_FLAGS_4(gemm, ...)` 之后）添加：

```c
DEF_HELPER_FLAGS_4(vadd, TCG_CALL_NO_WG, void, env, tl, tl, tl)
```

`vadd` 不写回寄存器（结果直接写入内存），返回 `void`，参数为 `(env, rd_addr, rs1_addr, rs2_addr)`，与 `gemm`/`vrelu` 等同模式。

### 3. `target/riscv/op_helper.c`（helper 实现）

在 `HELPER(gemm)` 函数之后添加 `HELPER(vadd)` 实现：

```c
void HELPER(vadd)(CPURISCVState *env, target_ulong rd,
                  target_ulong rs1, target_ulong rs2)
{
    for (int i = 0; i < 16; i++) {
        int32_t a = (int32_t)cpu_ldl_data(env, rs1 + i * 4);
        int32_t b = (int32_t)cpu_ldl_data(env, rs2 + i * 4);
        cpu_stl_data(env, rd + i * 4, (uint32_t)(a + b));
    }
}
```

使用 `cpu_ldl_data()`/`cpu_stl_data()` 访问 guest 内存（与已有 helper 一致）。`int32_t` 加法天然实现补码回绕。原地操作无需特殊处理，因为先读后写在逐元素粒度上是安全的。

### 4. `target/riscv/insn_trans/trans_xg233ai.c.inc`（翻译函数）

在 `trans_gemm()` 之后添加 `trans_vadd()`：

```c
static bool trans_vadd(DisasContext *ctx, arg_r *a)
{
    if (!ctx->cfg_ptr->ext_xg233ai) {
        return false;
    }

    TCGv rd = get_gpr(ctx, a->rd, EXT_NONE);
    TCGv rs1 = get_gpr(ctx, a->rs1, EXT_NONE);
    TCGv rs2 = get_gpr(ctx, a->rs2, EXT_NONE);

    gen_helper_vadd(tcg_env, rd, rs1, rs2);
    return true;
}
```

模式与 `trans_gemm` 完全一致：不写回 GPR，三个操作数都是内存地址，用 `get_gpr(..., EXT_NONE)` 读取。

## 不需要修改的文件

- **测试文件** `tests/gevico/tcg/riscv64/test-insn-vadd.c`：已存在，包含全部 5 个测试用例
- **构建配置** `tests/gevico/tcg/riscv64/Makefile.softmmu-target`：已包含 `insn-vadd` 在 `TEST_CASES` 中
- **扩展注册** `cpu_cfg_fields.h.inc`、`cpu.c` 等：`ext_xg233ai` 已注册，无需修改
- **Meson 构建** `target/riscv/meson.build`：decodetree 处理整个 `.decode` 文件，新增条目自动生效
