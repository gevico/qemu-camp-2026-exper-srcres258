# Plan: Implement `vscale` Instruction Translation for Xg233ai

## Summary

Implement the `vscale` (INT32 vector scalar multiply) instruction in QEMU's RISC-V TCG frontend. The instruction reads an INT32[16] array from source memory, multiplies each element by a scalar register value using INT64 intermediate precision, truncates to 32-bit, and writes results to destination memory. It supports in-place operation (source == destination).

## Current State

The Xg233ai extension has 6 instructions already implemented (`dma`, `sort`, `crush`, `expand`, `vdot`, `vrelu`). All extension infrastructure is in place. The test file `tests/gevico/tcg/riscv64/test-insn-vscale.c` already exists and is registered in the build system (`TEST_CASES` in `Makefile.softmmu-target`).

## Instruction Encoding

- funct7 = `1100110` (decimal 102)
- funct3 = `110`
- opcode = `1111011` (0x7B, custom-3)
- Format: `.insn r 0x7b, 6, 102, rd, rs1, rs2`

| Operand | Register | Meaning |
|---------|----------|---------|
| rd      | x[rd]    | Destination array base address |
| rs1     | x[rs1]   | Source array base address |
| rs2     | x[rs2]   | Scalar multiplier (register value, not address) |

The instruction does **not** write back to any GPR — it only performs memory-to-memory operations.


### 3. Helper Implementation: `target/riscv/op_helper.c`

Add after the existing `HELPER(vrelu)` (after line ~863). Implementation follows the pseudocode from the datasheet:

```c
void HELPER(vscale)(CPURISCVState *env, target_ulong rd,
                    target_ulong rs1, target_ulong rs2)
{
    int32_t scale = (int32_t)rs2;

    for (int i = 0; i < 16; i++) {
        int32_t val = (int32_t)cpu_ldl_data(env, rs1 + i * 4);
        cpu_stl_data(env, rd + i * 4, (uint32_t)((int64_t)val * (int64_t)scale));
    }
}
```

Key points:
- `cpu_ldl_data()` reads a 32-bit little-endian value (INT32).
- `cpu_stl_data()` writes a 32-bit little-endian value — the `(int64_t)val * (int64_t)scale` multiplication produces a 64-bit intermediate result, and casting to `uint32_t` performs the truncation to 32-bit as specified.
- Vector length is fixed at 16 elements.
- For in-place operation (rd == rs1), each element is read before being written, so no special handling is needed — the sequential access pattern ensures correctness.
- No GPR writeback needed.

### 4. Translation Function: `target/riscv/insn_trans/trans_xg233ai.c.inc`

Add `trans_vscale()` after `trans_vrelu()` (after line ~84). Follows the same pattern as `crush`/`expand`/`vrelu` — all operands use `get_gpr()` since this is a memory-only operation:

```c
static bool trans_vscale(DisasContext *ctx, arg_r *a)
{
    if (!ctx->cfg_ptr->ext_xg233ai) {
        return false;
    }

    TCGv rd = get_gpr(ctx, a->rd, EXT_NONE);
    TCGv rs1 = get_gpr(ctx, a->rs1, EXT_NONE);
    TCGv rs2 = get_gpr(ctx, a->rs2, EXT_NONE);

    gen_helper_vscale(tcg_env, rd, rs1, rs2);
    return true;
}
```

Key points:
- Uses `get_gpr()` for all three operands (no GPR writeback).
- `gen_helper_vscale(tcg_env, rd, rs1, rs2)` — all memory operations, no return TCGv.
